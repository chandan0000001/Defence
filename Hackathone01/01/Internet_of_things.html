HOW Does Internet Work..?

Principle 1: The Need to Share Information

The fundamental goal is to transfer information from one computer (A) to another (B). The simplest method is physical transfer (e.g., USB stick), but this is slow and inefficient. A better solution is to establish a direct electronic connection using a cable (ethernet or fiber optic). By encoding data as signals (high voltage = 1, low voltage = 0), the two computers can exchange bits. This forms the most basic type of network: a direct link between two computers.


Principle 2: The Problem of Scale

When multiple computers need to communicate, direct cabling between every pair quickly becomes unmanageable (scaling problem). The solution is a central device called a switch, where each computer connects only once. The switch receives data from one computer and forwards it to the intended recipient. This setup creates a Local Network (LAN), enabling efficient communication among nearby computers.

Principle 3: The Problem of Connecting Networks

When two local networks need to communicate, connecting every computer across both networks is impractical. The solution is a Router, a device that links entire networks and forwards traffic between them. By connecting routers from different networks, we can interconnect many networks together. This creates the Internet—a vast “network of networks,” built by linking countless local networks through routers.

Principle 4: The Problem of Finding Anyone (Addressing)

To send data across millions of computers worldwide, each device needs a unique identifier—an IP Address. Similar to postal addresses, IP addresses uniquely label every device on the Internet. Routers use these addresses to forward messages step by step,choosing the best "next hop" toward the destination. They do not need the full route, only the next direction to move the data closer to its target

Principle 5: The Problem of Reliability and Fairness

Sending large data as one stream is inefficient and error-prone. The solution is to split it into small, numbered packets, each containing data, source and destination IPs, and sequence info. Packets travel independently, possibly along different routes, and the receiver reassembles them in order, requesting missing ones if needed. These rules for breaking, sending, and reassembling data form a Protocol, with the primary one on the internet being TCP/IP.

IP addresses

An IPv4 (Internet Protocol version 4) address is the classic IP address format everyone is used to seeing.

  • Structure: It's a 32-bit number. To make it readable for humans, we divide it into four 8-bit sections, and write each section as a decimal number from 0 to 255.
    • Example: 172.217.16.142 (one of Google's addresses)
  • The Big Problem (The Why): A 32-bit number means there's a hard limit on the total number of possible addresses: 2^32, which is about 4.3 billion.

When the internet was designed, 4.3 billion seemed like an infinite number. But with the explosion of laptops, phones, servers, smart watches, and even smart refrigerators, we quickly realized we would run out. This scarcity is the single most important reason we have public and private IPs.

Public IP Address

A Public IP address is your global, unique address on the internet.

  • Analogy: The company's main, public phone number.
  • Uniqueness: It must be globally unique. No two devices on the internet can have the same public IP address at the same time.
  • Purpose: To be reachable from anywhere in the world. This is the address that web servers, email servers, and any other public service use.
  • Assignment: It is assigned to you by your Internet Service Provider (ISP) (Comcast, AT&T, etc.). You don't control it; you lease it from them.
  • Who has one? Your home router has one. The server hosting google.com has one. The server hosting Netflix has one.

When you go to a site like whatismyip.com, it shows you the Public IP address of your router. From the internet's perspective, your entire home network looks like a single device with that one public address

Private IP Address

A Private IP address is a local address used only within your own private network (e.g., your home Wi-Fi, your office network).

  • Analogy: The employee's private, 4-digit extension number.
  • Uniqueness: It only needs to be unique on your local network. My laptop can have the address 192.168.1.100 on my home Wi-Fi, and your laptop can have the exact same address on your home Wi-Fi. They don't conflict because they are in separate, private networks.
  • Purpose: To allow devices on the same local network to communicate with each other without needing a globally unique address for each one. Your laptop talks to your printer using private IPs. Your phone streams to your Chromecast using private IPs.
  • Assignment: It is assigned to your devices by your own router.
  • Reserved Ranges: The internet authorities have reserved specific ranges of addresses just for this purpose. If you see an IP address starting with one of these, you know it's a private address:
    • 10.0.0.0 to 10.255.255.255 (Used by large corporations)
    • 172.16.0.0 to 172.31.255.255 (Used by medium-sized networks)
    • 192.168.0.0 to 192.168.255.255 (Most common for home networks)

IPv6

The internet is running out of IPv4 addresses. To solve this, a new version of IP was created: IPv6.

  • Structure: It's a 128-bit number, written as eight groups of four hexadecimal digits (0-9 and A-F), separated by colons.
  • Example: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
  • Address Space: With 128 bits, IPv6 can support 2^128 addresses, which is about 340 undecillion addresses (that's 340 followed by 36 zeros). This is an unimaginably large number, enough to give every grain of sand on Earth its own unique IP address.
  • Adoption: IPv6 adoption has been slow but is steadily increasing. Many modern devices and operating systems support IPv6, but IPv4 is still widely used.

In summary, IPv4 and IPv6 are two versions of the Internet Protocol that define how devices on the internet are addressed and located. IPv4 is the older, more common version with a limited address space, while IPv6 is the newer version designed to accommodate the growing number of internet-connected devices with a vastly larger address space.

MAC Address:

>

A MAC Address (Media Access Control Address) is a unique 48-bit identifier permanently assigned to every network-capable device (like Wi-Fi cards, Ethernet ports, phones, or smart TVs). It’s usually shown in 12 hexadecimal digits and written in different formats for readability:

  1. Colon-separated → 3C:22:FB:A3:B4:C5
  2. Hyphen-separated → 3C-22-FB-A3-B4-C5
  3. Period-separated → 3c22.fba3.b4c5
  4. No separators → 3C22FBA3B4C5

Unlike IP addresses (which can change), MAC addresses are permanent hardware identifiers.

How to find your MAC Address:

Windows

1. Open Command Prompt:

  • Click the Start Menu.
  • Type cmd and press Enter.
  • (Alternatively, press Win + R, type cmd, and press Enter).
2. Run the Command

In the black window that appears, type the following command and press Enter:

ipconfig /all

If you only want to see the MAC addresses without all the other details, you can use the getmac command.

getmac /v

MACOS

1. Open Terminal:

  • Open Spotlight Search (press Cmd + Space).
  • Type Terminal and press Enter.
2. Run the Command

In the window that appears, type the following command and press Enter:

ifconfig

Port Number

A Port Number is a 16-bit identifier (0–65,535) used with IP addresses to route network traffic to the correct application or service on a device. Ports fall into three categories:

  1. Well-Known Ports (0–1023): Reserved for core internet services, managed by IANA, require admin rights
    • Examples: 21 (FTP), 22 (SSH), 25 (SMTP), 53 (DNS), 80 (HTTP), 443 (HTTPS).
  2. Registered Ports (1024–49151): Assigned to specific services by IANA, used by applications, do not require admin rights
    • Examples: 3306 (MySQL), 5432 (PostgreSQL), 6379 (Redis).
  3. Dynamic/Private Ports (49152–65535): Not assigned, used for temporary connections, chosen by the OS
    • Examples: Randomly assigned for client-side connections.
  4. Common Protocols and Their Ports:
    • HTTP: 80
    • HTTPS: 443
    • FTP: 21
    • SSH: 22
    • SMTP: 25
    • DNS: 53
    • MySQL: 3306
    • PostgreSQL: 5432
    • Redis: 6379

Together, IP addresses + port numbers ensure data reaches the right device and the right application on that device.

DNS(DOMAIN NAME SYSTEM)

DNS (Domain Name System) is the global “phonebook” of the internet, translating human-friendly domain names (like google.com) into machine-friendly IP addresses.

How DNS Works:

  1. Local Cache Check: Browser → OS → Router. If found, use it immediately.
  2. Recursive Resolver: If not cached, your ISP’s resolver (or a public one like Google 8.8.8.8, Cloudflare 1.1.1.1) is asked to find it.
  3. Hierarchical Lookup:
    • Root Servers: Direct the resolver to the right TLD servers (e.g., .com).
    • TLD Servers: Point to the authoritative servers for the domain (e.g., google.com).
    • Authoritative Server: Provides the actual IP (e.g., 142.250.72.206)
  4. Caching: Resolver stores the result (TTL) and sends it back to your OS and browser.
  5. Final Connection: Browser uses the IP + correct port (e.g., 443 for HTTPS) to connect to the website.

Without DNS, we’d have to memorize IP addresses instead of simple names.
