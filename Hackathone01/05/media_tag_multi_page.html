HTML MEDIA

First Principle: The Web is More Than Text and Images

Webpages should deliver diverse content, not just text and images. Previously, playing media required plugins like Flash or QuickTime, which were inefficient and inconsistent.

The Core Problem

Embedding video/audio in a standardized it is essential for universal browser compatibility.

Solution: HTML5 <video> and <audio> Tags

Dedicated tags for embedding media with built-in controls.

Part 1: Basics

Use the <video> tag with the controls attribute for a functional video player.


<iframe width="560" height="315"
    src="https://www.youtube.com/embed/NGYp1zCBthk"
    title="YouTube video player"
    frameborder="0"
    allow="autoplay"
    allowfullscreen>
</iframe>

Part 2: Attributes

  • width/height: Set dimensions.
  • autoplay: Starts playback automatically (requires muted).
  • loop: Restarts video after finishing.
  • poster: Displays an image before playback.

Part 3: <audio> Tag

Works like <video>, but for audio files.

Part 4: Advanced

  • Problem: Browser format incompatibility.
  • Solution: Use multiple <source> elements for fallback formats.
  • Accessibility: Add captions/subtitles with <track>.

Methods

  • Robust: Use <source> for maximum compatibility.
  • Simple: Use src for quick embedding (less reliable).

Comparison Table

Feature Example 1 (<source> Method) Example 2 (src Method)
Browser Compatibility Excellent. Provides multiple formats as fallbacks. Good, but not guaranteed. Relies on a single format.
Flexibility High. You can list many different formats. Low. Only one format is possible.
Simplicity More complex. Requires multiple lines and files. Very simple. Requires only one line.
Fallback for Old Browsers Excellent. Provides custom text if the <video> tag fails. None.
Best Use Case Public-facing websites where you need to support all users. Quick tests, internal projects, or when you are 100% certain your target audience supports your one format.

Conclusion: Which One Should You Use?

For public websites, prefer the <source> method for compatibility. The src method is simpler but riskier.

Creating the Copyright Symbol ©

1. Using the Entity Name (Recommended)

Code: &copy;

Example:

<footer>
    <p>Copyright © 2024 My Website.</p>
</footer>

2. Using the Entity Number

Code: &#169;

Example:

<footer>
    <p>Copyright &#169; 2024 My Website.</p>
</footer>

Other Common Symbol Entities

Use entity names or numbers for symbols like <, >, &, etc.

Symbol Description Entity Name Entity Number
© Copyright &copy; &#169;
® Registered Trademark &reg; &#174;
™ Trademark &trade; &#8482;
< Less-than (for showing code) &lt; &#60;
> Greater-than (for showing code) &gt; &#62;
& Ampersand &amp; &#38;
" Double Quote &quot; &#34;
€ Euro Sign &euro; &#8364;

Why use entities for ?

To display <p> as text, not as an HTML tag, use &lt;p&gt;.
